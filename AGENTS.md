# Documentation project instructions

## About this project

- This is a documentation site built on [Mintlify](https://mintlify.com)
- Pages are MDX files with YAML frontmatter
- Configuration lives in `docs.json`
- Run `mint dev` to preview locally
- Run `mint broken-links` to check links

## OpenAPI spec

The API reference is driven by `api-reference/openapi.json`. **Never edit it directly** — it is generated by `scripts/generate-openapi.js`:

```bash
node scripts/generate-openapi.js
```

**Source of truth is the API repo.** When `openmail` is checked out alongside this repo, the generator reads `../openmail/apps/api/src/openapi.json` (maintained by the API team) and writes it to both locations:

- `api-reference/openapi.json` — consumed by Mintlify for the API reference pages
- `../openmail/apps/api/src/openapi.json` — served live at `https://api.openmail.sh/openapi.json` (rewritten with normalized formatting)

**Only regenerate with the `openmail` repo present.** Without it, the generator falls back to the inline `spec` object in the script, which is not kept up to date — regenerating from the fallback silently removes newer endpoints from `api-reference/openapi.json`.

**Workflow for endpoint changes**: land the spec change in the `openmail` repo first, then regenerate here and commit the updated `api-reference/openapi.json`. Also add any new endpoints to the API reference nav groups in `docs.json` (e.g. `"POST /v1/pods/{id}/api-keys"`), or they won't appear in the sidebar.
